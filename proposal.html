1. Executive Summary

This proposal presents the design and implementation plan for an Inventory and Supply Chain Analytics System, a comprehensive database-driven application that manages products, suppliers, warehouses, purchase orders, sales orders, and shipments with real-time inventory tracking and analytics capabilities.

Problem Statement

Modern businesses struggle with inventory management challenges including stockouts, overstock situations, inefficient supplier management, and lack of real-time visibility into supply chain operations. Manual tracking methods are error-prone and fail to provide actionable insights for decision-making.

Proposed Solution

We propose developing a full-stack inventory management system with a Microsoft SQL Server backend that leverages advanced database features including stored procedures, triggers, views, and complex analytical queries. The system will feature a modern web-based frontend providing dashboards, CRUD operations, and comprehensive reporting capabilities. The primary focus is on demonstrating proficiency in SQL and database design while delivering a functional, real-world application.

2. Introduction

2.1 Background and Motivation

Effective inventory management is critical for business operations across industries. Organizations need systems that can track stock levels across multiple locations, manage supplier relationships, process orders efficiently, and provide analytical insights for strategic planning. This project aims to create such a system while demonstrating comprehensive understanding of database concepts learned throughout the Database Systems course.

The motivation behind choosing this domain includes:

  • Rich data relationships suitable for demonstrating relational database design
  • Natural use cases for stored procedures, triggers, and transactions
  • Opportunities for complex analytical queries using window functions and CTEs
  • Real-world applicability and professional relevance

2.2 Objectives

The primary objectives of this project are:

  1. Design a normalized relational database schema for inventory and supply chain management
  2. Implement stored procedures for encapsulating business logic within the database
  3. Create triggers for automated inventory updates and audit trails
  4. Develop views for simplified data access and reporting
  5. Demonstrate advanced SQL techniques including window functions, CTEs, and complex joins
  6. Build a user-friendly web interface for system interaction
  7. Implement proper transaction handling for data integrity

2.3 Scope and Limitations

In Scope

  • Product and category management
  • Supplier management with performance tracking
  • Multi-warehouse inventory tracking
  • Purchase order creation and receiving
  • Sales order processing and shipment tracking
  • Stock movement audit trail
  • Analytics dashboard and reports
  • User authentication and role-based access

Limitations

  • No integration with external ERP or accounting systems
  • No barcode/RFID scanning functionality
  • Single-currency support only
  • No mobile application (web-based only)

5.2 Key SQL Demonstrations

The following advanced SQL techniques will be demonstrated in this project:

5.2.1 Window Functions - Sales Trend Analysis

SELECT ProductID, FORMAT(OrderDate, 'yyyy-MM') AS MonthYear, SUM(Quantity * UnitPrice) AS MonthlySales, LAG(SUM(Quantity * UnitPrice)) OVER ( PARTITION BY ProductID ORDER BY FORMAT(OrderDate, 'yyyy-MM') ) AS PreviousMonth, SUM(SUM(Quantity * UnitPrice)) OVER ( PARTITION BY ProductID ORDER BY FORMAT(OrderDate, 'yyyy-MM') ) AS RunningTotal, RANK() OVER ( PARTITION BY FORMAT(OrderDate, 'yyyy-MM') ORDER BY SUM(Quantity * UnitPrice) DESC ) AS SalesRank FROM SalesOrders so JOIN SalesOrderItems soi ON so.SalesOrderID = soi.SalesOrderID GROUP BY ProductID, FORMAT(OrderDate, 'yyyy-MM')

5.2.2 Common Table Expressions (CTE) - Inventory Valuation

WITH InventoryValue AS ( SELECT w.WarehouseID, w.WarehouseName, c.CategoryID, c.CategoryName, SUM(i.QuantityOnHand * p.CostPrice) AS TotalValue FROM Inventory i JOIN Products p ON i.ProductID = p.ProductID JOIN Categories c ON p.CategoryID = c.CategoryID JOIN Warehouses w ON i.WarehouseID = w.WarehouseID GROUP BY w.WarehouseID, w.WarehouseName, c.CategoryID, c.CategoryName ) SELECT WarehouseName, CategoryName, TotalValue, SUM(TotalValue) OVER (PARTITION BY WarehouseID) AS WarehouseTotal FROM InventoryValue ORDER BY WarehouseName, TotalValue DESC

5.2.3 Transaction Handling - Order Processing

BEGIN TRANSACTION BEGIN TRY UPDATE Inventory SET QuantityOnHand = QuantityOnHand - @OrderQty, QuantityReserved = QuantityReserved - @OrderQty WHERE ProductID = @ProductID AND WarehouseID = @WarehouseID; INSERT INTO StockMovements (ProductID, WarehouseID, MovementType, Quantity, ReferenceType, ReferenceID) VALUES (@ProductID, @WarehouseID, 'OUT', @OrderQty, 'SO', @SalesOrderID); UPDATE SalesOrders SET Status = 'Shipped' WHERE SalesOrderID = @SalesOrderID; COMMIT TRANSACTION END TRY BEGIN CATCH ROLLBACK TRANSACTION THROW; END CATCH

5.2.4 ABC Analysis (Pareto Principle)

WITH ProductRevenue AS ( SELECT p.ProductID, p.ProductName, SUM(soi.Quantity * soi.UnitPrice) AS Revenue FROM Products p JOIN SalesOrderItems soi ON p.ProductID = soi.ProductID GROUP BY p.ProductID, p.ProductName ), RankedProducts AS ( SELECT *, SUM(Revenue) OVER (ORDER BY Revenue DESC) AS CumulativeRevenue, SUM(Revenue) OVER () AS TotalRevenue FROM ProductRevenue ) SELECT *, CASE WHEN CumulativeRevenue / TotalRevenue <= 0.80 THEN 'A' WHEN CumulativeRevenue / TotalRevenue <= 0.95 THEN 'B' ELSE 'C' END AS ABCCategory FROM RankedProducts
